# Class

## Concept
Allows to define a subclass for this feature to use other features, as to conform to other object models.

## Folder structure

```bash
├── class
│   ├── index.js
│   ├── features.js
```

- The **class** folder contains 
    - the es6 models handled by the feature.
    - the features declaration.

## Variant entry points
None

## Usage
### Declare a class

```js
export default ({ ParentClass }) => class TargetClass extends ParentClass {
    disposableOrphans() {
        const items = super.disposableOrphans ? super.disposableOrphans() : []
        return [
            ...items,
            'wallet'
        ]
    }

    disposableChildren() {
        const items = super.disposableChildren ? super.disposableChildren() : []
        return [
            ...items,
            'wallet'
        ]
    }
}
```
### Use features
```js
export default [
    //featureslist 
]
```
[See declaring features](../../guides/use.mdx)

## Example
We want our feature **walletable** to use disposableOrphans and disposableChildren to manage data consistency over deletion:

class/index.js
```js
export default ({ ParentClass }) => class TargetClass extends ParentClass {   

    disposableOrphans() {
        const items = super.disposableOrphans ? super.disposableOrphans() : []
        return [
            ...items,
            'wallet'
        ]
    }

    disposableChildren() {
        const items = super.disposableChildren ? super.disposableChildren() : []
        return [
            ...items,
            'wallet'
        ]
    }
}
``` 

class/features.js
```js
export default [
    "disposableorphansable",
    "disposablechildrenable"
]
```
